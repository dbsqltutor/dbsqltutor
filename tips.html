SQL Tip #1: Why SELECT * Hurts Performance More Than You Think

SELECT * looks harmless, but it forces the database to return every column — even the ones you don’t need. That means more I/O, more memory use, and slower queries, especially on wide tables.

Beginner

What’s really happening when you use SELECT *:

  • You return every column from every table — even unnecessary ones.
  • More data means more I/O, more memory, and more engine work.
  • Wide datatypes (JSON, XML, VARCHAR(MAX), BLOB) make the slowdown worse.
  • The optimizer may not use covering indexes, causing extra lookups.

A simple rule of thumb:
Only select the columns you actually need. It keeps queries faster and your intent clearer.

Instead of:
SELECT * FROM Orders

Try:
SELECT OrderID, OrderDate, TotalAmount FROM Orders

If you want to build better SQL habits, start by replacing SELECT * with the specific fields you need. It’s a small change that pays off immediately.
